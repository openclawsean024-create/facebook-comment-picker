# 【FB Lottery Tool】規格計劃書

## 1. 專案概述

### 1.1 專案背景與目的

LINE 抽獎活動是社群行銷的經典手法，但手動收集留言、隨機挑選中獎者不僅費時，還容易有人為操作空間。FB Lottery Tool 讓粉專管理者輸入貼文網址，系統自動抓取所有留言者名單，並提供多種公平公正的抽獎方式（純隨機、加權機率、關鍵字過濾）。抽獎結果可一鍵匯出中獎名單，大幅節省行銷人員的作業時間，並提升粉絲參與意願。
### 1.2 目標受眾（TA）

- 品牌行銷人員：LINE 抽獎活動頻繁，需快速統計參與者、抽出幸運得主
- 社群小編：同時管理多個 LINE 官方帳號，活動期間需快速處理抽獎
- 電商業者：節慶行銷必備工具，需要可驗證公正性的抽獎結果
### 1.3 專案範圍

In Scope：LINE 貼文留言收集（輸入網址/無需登入）、多元抽獎方式（隨機/加權/關鍵字過濾）、LINE 官方帳號 OAuth 驗證、抽獎結果匯出（中獎名單/存證）
Out of Scope：Facebook 留言收集（非目標平台）、多貼文同時抽獎、活動宣傳功能
### 1.4 參考網站分析

- Random Draw Tools：操作直覺但無 LINE 整合，需手動輸入名單
- Comment Picker（Facebook）：Facebook 限定，不支援 LINE
- LINE抽獎助手（第三方工具）：功能陽春，無加權抽獎、匯出功能

## 4. 前端功能規格

- LINE 官方帳號 OAuth 登入
- LINE 貼文 URL 輸入框（自動解析並驗證）
- 留言者名單自動抓取與列表顯示（暱稱/頭貼/留言時間）
- 過濾條件設定（關鍵字/時間區間/重複留言去除）
- 抽獎人數與抽獎方式設定（隨機/加權/關鍵字）
- 加權抽獎：依據互動次數設定權重（按讚/留言/分享各自加權）
- 抽獎結果動畫展示（滾動名單→定格中獎者）
- 中獎名單一鍵複製 / 匯出 CSV
- 歷史抽獎記錄（可回溯查詢）
## 5. 後端與技術規格

### 5.1 技術棧

- 前端：Next.js 14（App Router）+ Tailwind CSS
- LINE API：LINE Messaging API / LINE Login（OAuth 2.0）
- 後端：Next.js API Routes（無需獨立後端服務器）
- 資料庫：Supabase（PostgreSQL）+ Redis 快取
- 部署：Vercel
